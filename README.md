# Minimalist Agent Framework

An agent framework where tools are JSON definitions. Implementations are generated by an LLM on first use, then cached. Each tool runs inside a WebAssembly sandbox.

## How it works

1. **Tools** are declared as JSON files in `tools/`. Each file describes the tool's name, parameters, and sandbox constraints.
2. On startup, `SandboxBuilder` reads all tool definitions and compiles them into a single `.wasm` file via [Capsule](https://github.com/mavdol/capsule).
3. When the agent calls a tool, `Runner` checks for a cached implementation in `.cache/tool_impls.json`. If none exists, it asks the LLM to write a Python snippet and caches the result.
4. The implementation runs inside the Capsule WASM sandbox with the constraints declared in the tool's JSON (`allowed_files`, `allowed_hosts`, `ram`, `timeout`, etc.).

## Setup

```bash
pip install -r requirements.txt
python3 main.py
```

Environment variables:

| Variable | Default | Description |
|---|---|---|
| `OPENAI_BASE_URL` | OpenAI | Base URL for the LLM API |
| `OPENAI_MODEL` | `gpt-4o` | Model used for tool generation and agent |

## Defining a tool

Create a file in `tools/<name>.json`:

```json
{
    "name": "my_tool",
    "description": "What this tool does.",
    "parameters": {
        "type": "object",
        "properties": {
            "input": {
                "type": "string",
                "description": "The input value."
            }
        },
        "required": ["input"]
    },
    "capsule": {
        "compute": "LOW",
        "ram": "64MB",
        "timeout": "10s",
        "max_retries": 1,
        "allowed_files": [],
        "allowed_hosts": []
    }
}
```

Optional fields:

- **`hint`** — a plain-text instruction injected into the LLM prompt when generating the implementation. Useful when the LLM needs guidance on which API or approach to use.

```json
"hint": "Use the DuckDuckGo API at https://api.duckduckgo.com/?q={query}&format=json"
```

## Cache

| Path | Contents |
|---|---|
| `.cache/capsule_sandbox.py` | Auto-generated Python source for the sandbox |
| `.cache/capsule_sandbox.wasm` | Compiled WASM binary |
| `.cache/tool_impls.json` | LLM-generated tool implementations |
| `.cache/sandbox.hash` | Hash of tool definitions — used to detect when to recompile |

Delete `.cache/tool_impls.json` (or a single key inside it) to force regeneration of a tool's implementation.

## Built-in tools

| Tool | Description |
|---|---|
| `execute_code` | Runs arbitrary Python code |
| `read_file` | Reads a file from disk |
| `write_files` | Writes content to a file |
| `list_files` | Lists files in a directory |
